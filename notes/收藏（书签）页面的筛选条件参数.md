# 收藏（书签）页面的筛选条件参数

日期：2026-06-05

最近我发现收藏页面里出现了一些新的选项：最新或最早排序、年龄限制、作品标签、收藏时间。

查看默认请求的 URL 是：

```
https://www.pixiv.net/ajax/user/9460149/illusts/bookmarks?tag=&offset=0&limit=48&rest=show&order=desc&mode=all&lang=zh
```

下载器通过 `API.getBookmarkData` 请求该 URL，但现在 Pixiv 新增了一些参数。

参数列表：

| 参数     | 值                                  | 是否为新增参数 | 对应的筛选条件 | URL 里的参数                                                      |
| -------- | ----------------------------------- | -------------- | -------------- | ----------------------------------------------------------------- |
| tag      | 空值或字符串                        | x              | 收藏标签       | 默认无，仅在查询特定的收藏标签时作为路径的末端，如 `/artworks/原神` |
| rest     | show 或 hide                        | x              | 公开或匿名收藏 | 始终有 `rest=show` 或 `rest=hide`                                 |
| order    | desc 或 asc                         | ✨              | 最新或最早排序 | 默认无（视为 desc），仅当为 asc 时有 `order=asc`                  |
| mode     | all 或 safe 或 r18                  | ✨              | 年龄限制       | 始终有 `mode=<value>`                                             |
| work_tag | 仅在需要时才携带这个参数，字符串    | ✨              | 作品标签       | 默认无，仅在查询特定的作品标签时有 `work_tag=<标签名>`            |
| bm       | 仅在需要时才携带这个参数，如 202402 | ✨              | 收藏时间       | 默认无，仅在指定收藏时间时有 `bm=<年-月>`，需要去掉中间的横线 `-`                          |
| offset   | 0 或 limit 的倍数                   | x              | -              | -                                                                 |
| limit    | 48 或 30                            | x              | -              | -                                                                 |
| lang     | zh 或其他语言标记                   | x              | -              | -                                                                 |

其中：
- 有 6 个参数对应着 6 种筛选条件。
- `rest` 和 `mode` 始终在 URL 里有对应参数。其他 4 个参数都只在设定为特定值时才会有。
- `work_tag`、`bm` 参数仅在用户启用了对应的筛选条件时才需要携带。但实际上传递空值也是可以的，所以在请求 API 时可以始终携带。

## 排序

### ✨按最新收藏排序

这是默认的排序方式，参数是 `order=desc`。

### ✨按最早收藏排序

🔑这是会员专属功能。参数是 `order=asc`。

PS：有趣的是，按最新排序和按最早排序显示的作品数量是不一样的。按最新排序：30463，按最早排序：28131，相差 2322，差不多正好是已经被删除的作品的数量。实际上我找到的被删除的作品数量是2127。

## 是否公开

### 仅限公开收藏

默认值，参数是 `rest=show`。

### 仅限匿名收藏

参数是 `rest=hide`。

## 年龄显示

### ✨不限

参数是 `mode=all`。

### ✨仅限全年龄

参数是 `mode=safe`。

### ✨仅限R-18

参数是 `mode=r18`。

⚠️注意：R-18G 也包括在这个分类里。

## 收藏标签

默认是“不限”，此时 tag 参数是空的 `tag=`。

如果选择了某个标签，那么 tag 就会是这个标签，如原神的参数是 `tag=%E5%8E%9F%E7%A5%9E`。

## 作品标签

🔑这是会员专属功能。参数是 `work_tag=标签名`，例如 `work_tag=%E5%8E%9F%E7%A5%9E`。

⚠️注意：“收藏标签”和“作品标签”是互斥的，同时只能使用其中一个。

## ✨收藏时间

只能按预设的月份选择，不能自由设置起始和结束时间。参数如 `bm=202402`。
