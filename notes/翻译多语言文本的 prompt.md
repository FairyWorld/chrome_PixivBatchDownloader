# 中文语句翻译成多语言的需求

## 背景说明

这个项目是一个浏览器扩展程序 [Powerful Pixiv Downloader](https://github.com/xuejianxianzun/PixivBatchDownloader)。它是一个爬虫和下载器，用于从 Pixiv.net 这个网站下载插画、漫画、小说等内容。大多数用户在 PC 端的浏览器上使用它。它有很多设置项，还会显示日志和一些提示消息。我让你翻译的语句都是需要显示在网页上供用户查看的。

## 每条语句的格式

- 每条语句有一个 key，其 value 是 string[]。
- value 的数组里包含 6 条 string，第 1 条是原文，后 5 条是其他语言的翻译，按顺序分别是：繁体中文、英语、日语、韩语、俄语。
- 数组里的每条字符串都使用反引号 ` 包裹。
- 数组的最后一条语句后面需要添加逗号。这是 JS 语法里数组项后面的逗号，不要添加到语句里。
- 不要添加注释。

## 术语表

一些中文词语对应的英语翻译如下：

- `作品`（指 pixiv 上的投稿）翻译为`work`。
- `插画`翻译为`illustration`。
- `漫画`翻译为`manga`。
- `小说`翻译为`novel`。
- `动图`（指 pixiv 上的逐帧动画）翻译为`Ugoira`。
- `图片`（对 pixiv 上的图片的统称）翻译为`image`。
- `简介`（指作品的介绍信息）翻译为`description`。
- `抓取`（让这个下载器爬取作品数据）翻译为`crawl`。
- `收藏`（把作品添加到收藏夹，相当于书签）翻译为`bookmark`。
- `设定资料`翻译为`glossary`。
- `珍藏册`翻译为`collections`。
- `点击`翻译为`click`。因为这个下载器是为 PC 端网页设计的，所以`点击`指的是鼠标点击，而非点击屏幕的 tap。
- `下载记录`有两种情况。第一种情况：指这个扩展程序保存的自己的下载记录格式，此时应该翻译为`download record`。第二种情况：指浏览器的下载记录（通常语句里会提及“浏览器”），此时翻译为`download history`。
- `发布时间`、`发表时间`、`投稿时间`是相同的意思，应该翻译为`posting time`。
- `命名规则` 在不同语言里的对应翻译（这是按照翻译顺序保存的多语言数组）：['命名规则','命名規則','Naming rule','命名規則','명명 규칙','Правила названий',]

## 附加说明

- 需要注意遵守术语表。
- 用户大多都没有计算机和编程知识，所以翻译风格可以口语化或者偏日常一些，让用户易于理解。
- 如果中文语句里有 html 标签，翻译时需要原样保留。例如可能有 `<strong></strong>` `<a></a>` 标签等。
- 如果原语句里有 `<span class="key">关键字</span>` 形式的标记，那么在翻译后的语句里也要保持这个标签，并把“关键字”翻译成对应的语言。
- 中文的引号如 `“` 和 `”` 在翻译成英语时，使用英语的引号 `"`。
- 如果原语句里使用了 `\n` 或 `<br>` 来换行，在翻译时需要保持换行格式（而非把多行文字合并到一行）。

## 示例

输入：
`<span class="key">原创</span>作品`

输出：

```js
[
`<span class="key">原创</span>作品`,
`<span class="key">原創</span>作品`,
`<span class="key">Original</span> works`,
`<span class="key">オリジナル</span>作品`,
`<span class="key">오리지널</span> 작품`,
`<span class="key">Оригинальные</span> работы`,
]
```
