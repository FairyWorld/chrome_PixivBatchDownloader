# 小说里置换单词的细节

示例小说：
https://www.pixiv.net/novel/show.php?id=17868764

所属系列：
https://www.pixiv.net/novel/series/9114820

当用户设置了替换单词之后，Pixiv 会在本地存储空间保存配置。这份配置是完全储存在本地的，不会发送到服务器。如果清除了本地数据，替换就会失效。

Pixiv 在网页上显示小说内容时，会先显示原文，然后获取替换词再对原文进行替换，可以看到小说内容产生了变化。

这项数据的 key 例如：`novel_series_glossary_9460149`，其组成分成 2 部分：前面的 `novel_series_glossary_` 字符串是固定的，最后的数字是当前用户 ID。

数据示例：

```json
{
  // 系列小说的 id
  "9114820": {
    "words": [
      {
        // 替换项的 id
        "id": "494121",
        // 用户设置的替换词
        "word": "一"
      },
      {
        "id": "494122",
        "word": "二"
      }
    ],
    "loadTime": 1783592019521
  }
}
```

但是这里面没有替换前的单词，所以需要根据替换项的 id 去寻找替换前的单词。

这个 id 在系列的数据、小说的数据里都没有，必须通过系列的设定资料来获取：

https://www.pixiv.net/ajax/novel/series/9114820/glossary

在设定资料里遍历 `body.categories` 数组，再在每个数组项的 `items` 数组里查找：如果这一项的 id 与替换项的 id 相同，那么其 name 就是替换前的单词。

`items` 数组示例：

```json
[
    {
        "id": "494121",
        "seriesId": "9114820",
        "categoryId": "242642",
        "name": "1",
        "overview": "可以置换单词1",
        "coverImage": null,
        "detail": null
    },
    {
        "id": "494122",
        "seriesId": "9114820",
        "categoryId": "242642",
        "name": "2",
        "overview": "可以置换单词2",
        "coverImage": null,
        "detail": null
    }
]
```

这样，我们就知道了完整的替换数据：
- 把 `1` 替换成 `一`
- 把 `2` 替换词 `二`
